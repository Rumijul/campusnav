# Knowledge

- 2026-03-24 — In this GSD worktree environment (`.gsd/worktrees/*`) with Node `v25.8.1` + Vite `v7.3.1`, local Vite runtime commands can fail with environment/path-specific errors (`dependency optimization ... reading 'imports'` during `npm run dev:client`, and emitted chunk `fileName` path errors during `npm run build`). For slice execution inside auto-mode worktrees, rely on Vitest verification surfaces for UI logic unless runtime browser UAT is explicitly required in a non-worktree runtime environment.
- 2026-03-24 — For cross-floor directions, compute connector `up/down` language from resolved floor numbers (using `floorMap` when available, with `floorId` fallback) and carry `floorId` + `floorNumber` on every `DirectionStep`. In UI rendering, group by contiguous floor runs and only show floor headers when there is more than one section. This avoids incorrect vertical wording when IDs are not ordered and preserves flat rendering for single-floor routes.
- 2026-03-24 — In Hono route handlers, `c.json(payload, status)` is typed against a literal `ContentfulStatusCode` union; if status comes from a custom error class, type that status as a literal union (e.g. `400 | 404`) instead of `number` to avoid TS2769 under strict typecheck.
- 2026-03-24 — `NavNode` connector fields are optional, and reducer-style `{ ...node, ...changes }` updates cannot reliably clear them to “unset” under strict optional typing. For server-driven connector unlink patches, rebuild the node object without connector keys (strip + conditional re-add) and replace full node entries instead of merging partial updates.
- 2026-03-24 — For admin connector linking UI, treat `/api/admin/connectors/link` as server-authoritative and patch both active-floor editor state and cached multi-floor `navGraph` from `updatedNodes`. Do not attempt local optimistic reciprocal edits; they drift on relink/unlink stale-link cleanup paths.
- 2026-03-24 — GPS floor bounds must be handled as an atomic tuple: accept either all four numeric values (`minLat`, `maxLat`, `minLng`, `maxLng`) or all four nulls to clear; reject partial tuples (`GPS_BOUNDS_INCOMPLETE`) and invalid ordering (`BOUNDS_RANGE_INVALID`) before persistence.
- 2026-03-24 — Keep `/api/map` floor `gpsBounds` complete-only and patch editor graph state from `PUT /api/admin/floors/:id/gps-bounds` responses by `floorId` (including campus floor). This prevents partial calibration drift and gives S27 a trustworthy projection contract.
- 2026-03-25 — In shell verification commands that use `awk` inside a quoted `bash -lc '...'` string, escape field variables as `\$2` (not `$2`), otherwise the shell expands `$2` before awk sees it and checkpoint-hash extraction silently degrades to a wrong value.
- 2026-03-25 — When `useGeolocation` resolves browser APIs internally, memoize the resolved adapter (`options.geolocation ?? resolveNavigatorGeolocation()`) before using it as an effect dependency; recreating adapter objects each render can cause repeated watch restarts and React maximum-update-depth noise in browser runs.
- 2026-03-25 — For Konva overlay component tests in this codebase (e.g., GPS marker/ring layers), assert React element props (`Circle` name/x/y/radius/strokeWidth) directly instead of DOM queries; this keeps tests deterministic without requiring canvas rendering harnesses.
- 2026-03-25 — For hook-heavy UI overlays that are awkward to mount in jsdom (like SearchOverlay geolocation states), `renderToStaticMarkup` plus explicit text/attribute assertions is a reliable way to verify fallback copy, button disablement, and manual-control continuity.
- 2026-03-25 — In Konva multi-touch handlers, compute stage-local touch midpoint from `stage.getAbsoluteTransform().copy().invert().point(...)` **before** mutating stage state, then apply scale → rotation → position recalculation in that order. Re-introducing separate midpoint `dx/dy` pan deltas double-counts movement and causes focal drift.
- 2026-03-25 — Milestone closeout quality gates depend on slice-level `S##-SUMMARY.md` artifacts. Task-level summaries alone are not enough for a clean audit verdict; backfill missing slice summaries proactively to avoid `needs-attention` closeouts.
