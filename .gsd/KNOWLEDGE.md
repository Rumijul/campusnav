# Knowledge

- 2026-03-24 — In this GSD worktree environment (`.gsd/worktrees/*`) with Node `v25.8.1` + Vite `v7.3.1`, local Vite runtime commands can fail with environment/path-specific errors (`dependency optimization ... reading 'imports'` during `npm run dev:client`, and emitted chunk `fileName` path errors during `npm run build`). For slice execution inside auto-mode worktrees, rely on Vitest verification surfaces for UI logic unless runtime browser UAT is explicitly required in a non-worktree runtime environment.
- 2026-03-24 — For cross-floor directions, compute connector `up/down` language from resolved floor numbers (using `floorMap` when available, with `floorId` fallback) and carry `floorId` + `floorNumber` on every `DirectionStep`. In UI rendering, group by contiguous floor runs and only show floor headers when there is more than one section. This avoids incorrect vertical wording when IDs are not ordered and preserves flat rendering for single-floor routes.
- 2026-03-24 — In Hono route handlers, `c.json(payload, status)` is typed against a literal `ContentfulStatusCode` union; if status comes from a custom error class, type that status as a literal union (e.g. `400 | 404`) instead of `number` to avoid TS2769 under strict typecheck.
- 2026-03-24 — `NavNode` connector fields are optional, and reducer-style `{ ...node, ...changes }` updates cannot reliably clear them to “unset” under strict optional typing. For server-driven connector unlink patches, rebuild the node object without connector keys (strip + conditional re-add) and replace full node entries instead of merging partial updates.
