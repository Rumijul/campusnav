## Gaps by Category

### Orphaned Requirements (Phase Missing VERIFICATION.md)

**Context:** These requirements are functionally built and human-UAT approved. The gap is documentation-only — no formal VERIFICATION.md was created for Phases 7, 8, or 9.

- **ADMN-02** (Phase 7): GET /api/map is public, SQLite DB serves student app — confirmed in integration check. Plan 07-04 (human verification) was never run. VERIFICATION.md was never created.
- **ADMN-01** (Phase 8): JWT login flow built and human-approved in 08-03 SUMMARY. No VERIFICATION.md created.
- **EDIT-01–05** (Phase 9): Full admin visual editor built and human-approved in 09-04 SUMMARY. No VERIFICATION.md created.

**Resolution options:**
- Run `/gsd:verify-work 7`, `/gsd:verify-work 8`, `/gsd:verify-work 9` to retroactively create VERIFICATION.md files
- Or accept as tech debt and complete milestone if human UAT SUMMARYs are sufficient evidence

### ROUT-07 Mechanism Change

**Issue:** Phase 4 verified ROUT-07 ("tap to see location details") with a full LandmarkSheet (snap points, all detail fields). Phase 5 removed LandmarkSheet from the render tree — landmark tap now feeds route selection instead of opening a detail panel. Full detail fields (roomNumber, description, buildingName, accessibilityNotes) are no longer viewable by the student.

**Impact on users:** Students cannot browse location details independently — they can only see location labels in route flow. This was an intentional product decision (verified by Phase 5 UAT approval) but represents a functional reduction from the original ROUT-07 specification.

**Resolution options:**
- Accept the changed behavior (Phase 5 UAT approved the whole flow)
- Reintroduce a detail panel as a gap-closure phase

### Data Tab Visibility (EDIT-07, EDIT-08)

**Issue:** UAT for Phase 10 noted "Data tab not visible in UI." Code analysis confirms the implementation is structurally correct (tab switcher, className toggling, flex layout chain). Root cause not determined during UAT. Deferred.

**Likely fix:** Add `min-h-0` to the root flex container in `MapEditorCanvas.tsx` (common Tailwind flex-col height issue).
