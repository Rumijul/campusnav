**Test:** In the admin editor, place a node, switch to select mode, click the node (it should select and show the side panel), then click the same node again.

### 2. Floor Plan Image Click Deselect in Admin Map Editor

**Test:** Select a node in the admin editor (side panel opens), then click the floor plan image background (not any node or edge).
**Expected:** Selection clears, side panel closes.
**Why human:** Konva event target discrimination (`e.target` vs stage vs image shape) requires live canvas interaction to confirm the image shape click is properly distinguishable.

### 3. A/B Pin Tap Clears Endpoint (Touch Device)

**Test:** On a touch device or browser device emulation, set a start and destination to make both pins visible, then tap the A (green) pin.
**Expected:** The A pin disappears, the search overlay shows an empty start field with 'start' as the active field, and the route line disappears.
**Why human:** Touch event handling (`onTap` vs `onClick`), `cancelBubble` effectiveness against `LandmarkLayer`, and route clearing require live interaction to confirm.

### 4. Room # Inline Edit in Data Table

**Test:** In the admin Data tab, click the Room # cell for a node that has no room number (shows "—"), type "204", press Enter.
**Expected:** Cell exits edit mode and shows "204". Save the graph and reload — the room number persists.
**Why human:** The inline edit state machine, blur/Enter behavior, and API persistence require live browser interaction.
