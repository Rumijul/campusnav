# Phase 14.1: node-selection-fixes-and-admin-room-number-edit - Context

**Gathered:** 2026-02-28
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix two categories of bugs/gaps in the admin map editor and student-facing app:

1. **Node selection bugs** — toggle deselect in the admin canvas, and floor-plan-image click deselects selection
2. **Admin room number editing** — the data table `Room #` column currently displays room numbers but is not editable inline; add inline editing consistent with Name and Type

The student-facing selection fix (A/B pin tap-to-deselect) is also in scope as it is part of the "node selection" theme named in the phase title.

</domain>

<decisions>
## Implementation Decisions

### Admin editor — node selection toggle
- Clicking an **already-selected node** in select mode should **deselect it** (toggle): `selectedNodeId` → `null`, side panel closes
- Clicking the **floor plan image** in select mode should also deselect (currently only transparent stage area deselects)
- Clicking an edge already deselects any selected node — keep this behavior

### Student app — A/B pin tap-to-deselect
- The A and B pins in `SelectionMarkerLayer` should be **tappable/clickable**
- Tapping the A pin clears `start` (calls `clearStart`)
- Tapping the B pin clears `destination` (calls `clearDestination`)
- After clearing: active field follows the cleared slot — clearing A sets `activeField` to `'start'`, clearing B sets `activeField` to `'destination'`
- Route disappears when either slot is cleared (already handled by existing `useEffect` in `FloorPlanCanvas`)

### Admin data table — room number inline edit
- Add inline editing to the **Room # column** in `NodeDataTable`, consistent with Name and Type inline edit pattern already present
- Behavior: click the room number cell to enter edit mode; `Enter` or blur commits; `Escape` cancels (or blur commits — match existing pattern)
- **Empty string treated as `null`** (no room number) on commit
- **Free text, no validation** — anything goes ('204', '2A', 'B-104')
- **Not sortable or filterable** — just editable; no sort header on Room # column
- Side panel room number field already works and stays as-is

### Room number search / display
- Search already handles `roomNumber` in `useLocationSearch` — no changes needed to search logic
- `LocationDetailSheet` already shows room number when present — no changes needed there
- **Search result dropdown**: add room number display alongside the room name in `SearchOverlay` result items
  - Show as secondary text, e.g. "Science Lab · Room 204" or similar subtle sub-label
  - Only show if `node.roomNumber` is present

### Claude's Discretion
- Exact styling of room number in search result items (badge, bullet separator, small secondary text)
- Whether to use existing `editingCell` state shape in `NodeDataTable` or add `roomNumber` to the union type
- How `FloorPlanImage` click events propagate to trigger deselect in admin canvas (may need `onClick` on the Konva image shape or layer)

</decisions>

<specifics>
## Specific Ideas

- The inline edit pattern in `NodeDataTable` for `label` and `type` (click span → input appears → blur or Enter commits) should be replicated exactly for `roomNumber`
- The A/B pins becoming tappable should feel natural — same tap-target size as the existing landmark markers (hit area expansion via `hitFunc` if needed)

</specifics>
